# 整体架构

## 服务

项目当前整体具有五个核心服务。

<Callout type='warning'>
	由于还在初步开发期间，服务架构随时可能产生较大的变化。
</Callout>

- `web`：前端。
- `api`：负责处理所有核心以及基础请求，包括文档创建、删除，专栏创建、删除等等。
- `celery-worker`：负责处理所有后台任务，包括通知发送、向量化、转码、数据同步、数据清理等。
- `file-backend`：文件服务器，存储所有文档信息和专栏信息。
- `daily-hot`：热搜聚集服务，注意本服务完全继承于[DailyHotApi](https://github.com/imsyy/DailyHotApi)，感谢原作者[imsyy](https://github.com/imsyy)的付出。

## 打包

项目使用 `docker-compose` 进行打包，打包配置位于 `docker-compose.yaml` 中。
