# 文档

本系统支持多类型文档来源，当前支持如下：网站文档，速记文档，文件分析。

## 文档状态

每个文档具有独立的状态，你可以在已读、未读状态之间自由切换。同时支持文档的收藏以及打标功能，方便你后续分类和查看。

默认情况下，新文档的状态都是`未读`（包括自己编辑的速记文档）。

## 文档查看

支持不同状态的文档单独查看，也支持查看所有文档。你也可以通过标签来筛选。

支持文档标题和内容的精准搜索（基于mysql查询），也支持全局模糊搜索（基于milvus）。

支持时间排序。

## 文档存储

当前文档存储服务是file-backend在负责，文档内容会存储在本地，同时会基于Embedding技术存储向量化的文档到milvus中，方便后续的全局搜索以及AI交互。

## 智能总结

你可以通过配置模型（设置页面内）来让系统支持自动总结文档内容，并替换掉默认的网站标题、描述等内容。

![](https://qingyon-revornix-public.oss-cn-beijing.aliyuncs.com/images/202507031531236.png)

## 补充笔记

你可以为文档添加补充笔记，方便你后续再查看的时候浏览。

## 文档收录

![](https://qingyon-revornix-public.oss-cn-beijing.aliyuncs.com/images/202507021304606.png)

有多种收录文档、资讯的方式，其中最简单的方式也是最直白的方式就是通过网站端的增加文档。

### 网站端收录

#### 配置说明

- 链接 你想要收录的网站链接
- 标签 你的文档标签，可以添加多个，方便后续筛选查看
- AI 自动总结 开启后，系统会自动调用 AI 接口，将文档内容进行总结，并替换掉默认的网站标题、描述等内容。
- 推送专栏 可以选择你希望推送的专栏，系统会自动将文档推送到你选择的专栏中，并更新该专栏的总结。

### API 收录

当前版本已开放 API 收录文档，你可通过点击应用页面内左下角用户信息->账户进入账户配置页，选择其中的开发 APIKey 配置前往配置。

详情请求参数和返回范例见[三方API](./api)
