# 专栏

<Callout type='info'>
	专栏功能当前属于Beta阶段，功能还极不稳定，后续可能发生较大变动。
</Callout>

设计专栏功能的目的是为了能够更方便的集成知识库，方便用户进行**知识总结、吸收以及分享**。

同时订阅功能也更好的支持了知识库的传播。

后续将会推出**云版本**，在云版本中会存在付费订阅，此设计的目的是为了提供给知识产出变现，用户可以付费订阅专栏，作者也可以通过他人的订阅**获得收益**，本平台将会收取 **10%** 的服务费。

## 今日专栏

这是一个特定的专栏类型，不需要用户进行配置，自动会生成，用户只需要向Revornix添加文档即可。即：每当用户于新的一天创建第一份文档的时候，系统会自动创建一个今日专栏，并将文档添加到今日专栏中，然后进行总结分析。

核心目的是为了将知识日期化整理，便于用户进行知识回顾。

专栏和**通知功能**是有一定绑定的，我也特地为当日总结做了一份通知模版，你可以在通知任务页面内直接配置该模版，这样用户在自己设定的时间点就会收到当天的总结。

## 文档绑定

专栏可以视为一种**特定知识集合**，我对专栏的设置理念是便于知识的**总结、吸收与分享**。

所以这一功能和文档是高强度绑定的，文档绑定专栏后，文档将自动出现在专栏中，专栏将自动出现在文档中。

_后续专栏会内嵌GraphRag，便于了解知识集合的节点图谱。_

## 分享与订阅

专栏可以设置为公开，公开的专栏用户可以在专栏社区内浏览到，并且可以进行订阅。

_后续将会设计为订阅的专栏发生更新后订阅用户会收到通知。_

在**云版本**中，将会有付费专栏的逻辑，用户可以通过将自己的专栏设置为付费浏览来进行知识变现，同时Revornix云平台将会收取 **10%** 的服务费用。

## 专栏查看

支持专栏标签筛选。

支持用户查看自己的专栏、社区的专栏以及自己订阅的专栏。

支持时间排序。

支持专栏标题精准搜索。

## 交流机制

专栏支持评论，用户可以在专栏右侧发表评论。

## 创建

当前创建专栏的方式仅有一种，即通过网站端创建，暂不支持 API 创建。

### 网站端创建

进入清链应用后，在左侧展开的 sidebar 的专栏两字右侧即有创建入口，点击即可进入创建页面。

#### 配置说明

| 配置项   | 说明                                                             |
| -------- | ---------------------------------------------------------------- |
| 图片     | 每个专栏都必须有一个封面图                                       |
| 专栏名称 | 专栏的名称                                                       |
| 专栏描述 | 专栏的描述                                                       |
| 专栏标签 | 专栏的标签，不仅方便自己筛选，更是能够增加在社区被推荐的概率     |
| 是否公开 | 是否公开，默认私有，如果设为公开，该专栏及其相关文档将在社区可见 |
